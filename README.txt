ДЛЯ РАБОТЫ С БД НЕОБХОДИМО:
==========================================

1. PostgreSQL на компьютере
   - Создать пользователя (по умолчанию postgres)
   - Выбрать пароль
   - Настроить IP-адрес (по умолчанию 127.0.0.1)
   - Выбрать ХОСТ (по умолчанию 5432)

2. Скачать библиотеки, указанные в requirements.txt

3. Позже с фронтэндом будет добавлен .bat файл для запуска

ПОРЯДОК НАПОЛНЕНИЯ СИСТЕМЫ СКУД (АЛГОРИТМ)
==========================================

ШАГ 1: БАЗОВЫЕ СПРАВОЧНИКИ (Фундамент)
--------------------------------------
Эти данные не зависят от других таблиц. Без них нельзя двигаться дальше.

1. ДОБАВИТЬ РОЛИ (/admin/add_role)
   - Сначала создаем "Лаборант", "Преподаватель", "Админ".
   - Запоминаем полученный role_id (например, 1).

2. ДОБАВИТЬ КОМНАТЫ (/admin/add-room)
   - Создаем помещения: "Кабинет 101", "Лаборатория №1".
   - Запоминаем полученный room_id (например, 1).

3. ДОБАВИТЬ ГРУППЫ ДОСТУПА (/admin/add-access-group)
   - Создаем пустые названия групп: "Допуск в ЛБ-1", "Общий доступ".
   - Запоминаем полученный group_id (например, 1).

ШАГ 2: ИНФРАСТРУКТУРА И ПРАВА (Связи объектов)
----------------------------------------------
Настраиваем логику: какая дверь куда ведет и какие комнаты в каких группах.

4. ДОБАВИТЬ ТОЧКИ ДОСТУПА (/admin/add-access-point)
   - Привязываем физическую дверь к комнате (используем room_id).
   - Теперь система знает, что Считыватель №5 стоит на входе в Комнату №1.

5. НАПОЛНИТЬ ГРУППЫ КОМНАТАМИ (/admin/assign-room-to-group)
   - Кладем "ключи" от комнат в группы (group_id + room_id).
   - Теперь Группа №1 официально открывает Комнату №1.


ШАГ 3: ПОЛЬЗОВАТЕЛИ И КЛЮЧИ (Персонализация)
--------------------------------------------
Регистрируем людей и выдаем им права.

6. ДОБАВИТЬ СОТРУДНИКА (/admin/add-employee)
   - Регистрируем Пяткова Д.А.
   - Обязательно указываем role_id (созданный на шаге 1).

7. ВЫДАТЬ ГРУППУ СОТРУДНИКУ (/admin/assign-group-to-employee)
   - Самый важный момент для лаборантов.
   - Связываем employee_id Пяткова с его персональной группой group_id.
